Начнем урок со знакомства с языком программирования GDScript.

GDScript Это встроенный язык программирования в игровом движке Godot, специально разработанный для создания игр.

Он похож на Python, что делает его доступным для новичков и мощным для опытных разработчиков. GDScript имеет простой и интуитивно понятный синтаксис. Вот несколько ключевых конструкций, которые вам нужно знать:

  • Комментарии используются для пояснения кода. В GDScript есть однострочные и многострочные комментарии. Комментарии помогают разработчикам лучше понимать код, особенно если к проекту возвращаются спустя некоторое время или если код читают другие разработчики.
# Это однострочный комментарий

"""
Это
многострочный
комментарий
"""

В GDScript, как и в Python, отступы важны. Они определяют блоки кода. Правильное использование отступов помогает структурировать код и делает его более читаемым. Неправильные отступы могут привести к ошибкам выполнения.

  • Условные операторы позволяют выполнять код в зависимости от условий. Они являются основным инструментом для принятия решений в программе. Условные операторы помогают создавать более сложные логические структуры и управлять потоком выполнения программы.
if x > 10:
    print("x больше 10")
elif x == 10:
    print("x равно 10")
else:
    print("x меньше 10")
  • Циклы используются для повторения выполнения блока кода. Они позволяют выполнять одни и те же действия многократно, что особенно полезно при работе с массивами или при необходимости выполнения повторяющихся задач.
for i in range(5):
    print(i)

while x < 10:
    x += 1
    print(x)

Работа с переменными и типами данных

Переменные Это именованные контейнеры для хранения данных. В GDScript переменные могут быть объявлены и инициализированы следующим образом. 
var my_variable = 10
var my_string = "Hello, Godot!"

GDScript поддерживает несколько основных типов данных. Каждый тип данных имеет свои особенности и предназначен для хранения определенного вида информации. Правильное использование типов данных помогает оптимизировать работу программы и избежать ошибок:

  • int: Целые числа
  • float: Числа с плавающей точкой
  • String: Строки
  • bool: Логические значения (true/false)
  • Array: Массивы
  • Dictionary: Словари
var my_int = 42
var my_float = 3.14
var my_bool = true
var my_array = [1, 2, 3]
var my_dict = {"key": "value"}

Иногда нужно преобразовать один тип данных в другой. Преобразование типов позволяет адаптировать данные для различных операций и избежать ошибок, связанных с несовместимостью типов

var Number = 10
var string_Number = str(Number) # Преобразование int в String
var Float_Number = float(Number) # Преобразование int в float
var string = “12”
var Number_string = int(String) # Преобразование String в int

Функции и методы

Функции Это блоки кода, которые выполняют определенные задачи и могут быть вызваны по имени. В GDScript функции объявляются с помощью ключевого слова func.

Функции помогают структурировать код, делая его более модульным и удобным для повторного использования.

func my_function():
    print("Это моя функция")

Функции могут принимать параметры и возвращать значения. Это позволяет передавать данные в функции и получать результат их работы, что делает функции более гибкими и мощными.

func add_numbers(a, b):
    return a + b

var result = add_numbers(5, 3)  # result будет равно 8

Godot предоставляет множество встроенных функций для работы с игровыми объектами, сценами и ресурсами. Например, функция print() выводит текст в консоль. Встроенные функции облегчают выполнение часто используемых задач и позволяют сосредоточиться на создании уникальных игровых механик.

print("Hello, Godot!")

Как видите, GDScript своим синтаксисом очень сильно напоминает Python и, хотя они не являются родственниками, их сходство облегчает создание игр как для новичков, так и для гуру-программистов!

Теперь перейдем к самому интересному — добавлению игрового персонажа! Так как в Godot добавление игровых объектов, а также их свойств осуществляется через узлы, то и структура вашего героя будет иметь вид древа: основной узел героя будет содержать в себе дочерние узлы – спрайт (внешний вид) персонажа, его анимации, область коллизии (столкновения), камера персонажа и прочее.

Итак, создадим главный узел игрока — добавьте в вашу сцену новый узел и выберите тип CharacterBody2D. Добавить узел можно двумя способами:

  • Нажать ПКМ по сцене и выбрать пункт Добавить узел сюда
  • Нажать ПКМ по узлу сцены в древе и выбрать пункт Добавить дочерний узел

Результатом любого из способов будет открывшееся окно, как на рисунке 1.

В открывшемся окне, как показано на рисунке 1 необходимо ввести в поле поиска имя узла — CharacterBody2D и нажать ЛКМ кнопку Создать.

CharacterBody2D Это специальный узел в Godot (начиная с версии 4), предназначенный для управления 2D-персонажами и объектами, которые должны самостоятельно контролировать свое движение и столкновения.

На сцене (рис. 2) появится перекрестие, обозначающее местоположение вновь созданного узла, также, в древе сцен и узлов (рис. 3) появится новый узел.

Обратите внимание! Сейчас узел выдает предупреждение о том, что узел не имеет формы (рис. 3) и, соответственно, не умеет сталкиваться с другими объектами. Но в дальнейшем мы добавим другие дочерние объекты и данное предупреждение исчезнет!
Также! Напоминаем! Что всем узлам и сценам рекомендуется давать осмысленные имена, чтобы быстро находить нужные вам объекты в древе сцен!

Дайте имя созданному узлу, например, Player или MainHero. Имя можно выбрать любое. Данный узел станет Корневым для вашего героя. Все дальнейшие узлы будут дочерними.

Теперь добавим изображение герою. Для этого нажмите ПКМ по корневому узлу героя и добавьте ему дочерний узел Sprite2D (рис. 4).

Sprite2D Это узел Godot (начиная с версии 4), предназначенный для отображения двумерного изображения (текстуры) в сцене. Обычно он используется, чтобы визуально представить персонажей, объекты окружения, эффекты и т.п.

Нажав ЛКМ по узлу Sprite2D, обратите внимание на окно Инспектора (рис. 5).

В параметре Texture – текстура необходимо загрузить из ассетов проекта изображение героя. По умолчанию это файл tile_0002.png, однако, вы можете выбрать по своему желанию любой другой файл (рис. 6).

После чего на вашей сцене появится ваш герой (рис. 7).

Следующим важным дочерним узлом вашего персонажа является — CollisionShape2D.

CollisionShape2D Это узел в Godot, который определяет форму для столкновений (коллизий) в двухмерном пространстве. Проще говоря, он указывает движку, где именно находится «физическая граница» объекта.

Нажмите ПКМ по корневому узлу героя и добавьте ему дочерний узел CollisionShape2D (рис. 8).

В окне Инспектора данного узла выберите для параметра Shape значение Rectangle (прямоугольник) (рис. 9). И отредактируйте границы прямоугольника под границы спрайта героя, как показано на рисунке 8.

Обратите внимание на параметр Transform в Инспекторе. У корневого узла и дочерних, у дочерних объектов параметры x и y должны быть (0,0), иначе «физическое тело» будет в одном месте, его изображение в другом, а область столкновения может быть вообще в третьем (рис. 9).

Перемещать персонажа вместе со всеми его узлами сразу можно, используя инструмент Перемещение (горячая клавиша W). Главное выделить корневой узел – CharacterBody2D.

Важно! Не забудьте после перемещения вернуться в режим Выделение (горячая клавиша Q)

Последним узлом, который необходимо добавить герою, является узел Camera2D.

Camera2D Это специальный узел в Godot, который определяет, какую часть 2D-сцены видит игрок на экране. Если кратко, он «смотрит» на сцену и выводит выбранную область на экран.

С помощью параметра Zoom в Инспекторе можно настроить масштаб камеры, приближая или отдаляя его относительно главного героя.

Теперь прикрепите скрипт к корневому узлу вашего героя. Наш скрипт будет наследоваться от CharacterBody2D и позволит перемещать персонажа по всем 4 направлениям. Чтобы прикрепить новый скрипт, выделите ПКМ корневой узел героя – MainHero. И выберите Прикрепить скрипт (рис. 11).

Появится новое окно – прикрепить скрипт узла (рис. 12).

Убедитесь, что в параметре Наследует выбран корневой узел героя – CharacterBody2D. Обязательно отключите пункт параметра Шаблон и убедитесь, что скрипт будет сохранен в папку Scripts. После чего нажмите кнопку Создать.

После чего откроется встроенный редактор кода (рис. 13). Вы можете также использовать внешние редакторы кода, например – Visual Studio Code, но в данную среду необходимо будет добавить дополнение, например godot-tools от разработчика Geequlim. Поэтому отчасти проще использовать встроенный редактор.

Начнем создание скрипта!

extends CharacterBody2D

var speed = 200

Разберем данные строки:

  • extends CharacterBody2D – данная строка говорит, что данный скрипт наследуется от узла CharacterBody2D. Это значит, что скрипт сможет использовать все свойства и методы, связанные с движением и физикой такого узла.
  • var speed = 200 — Объявление переменной speed со значением 200. Отвечает за скорость передвижения персонажа по осям X и Y.

 Продолжим создание скрипта:

extends CharacterBody2D

var speed = 200

func _physics_process(_delta):
    if Input.is_action_pressed("ui_right"):
        velocity.x = speed
    elif Input.is_action_pressed("ui_left"):
        velocity.x = -speed
    else:
        velocity.x = 0

Разберем код построчно:

  • func _physics_process(_delta): — в данной строке происходит определение функции _physics_process(), это функция, которую Godot вызывает через равные промежутки времени, в соответствии с настроенной частотой физического шага (по умолчанию 60 раз в секунду). Она предназначена для обработки логики, зависящей от физики и стабильного тайминга.

  • Аргумент _delta содержит время (в секундах), прошедшее с момента прошлого кадра. Важно использовать _delta в расчетах движения т.к. это обеспечивает одинаковую скорость независимо от частоты кадров.

  • Далее идет условная конструкция. Input.is_action_pressed считывает нажатие определенной в скобках клавиши. В данной условной конструкции идет проверка клавиш «ui_right« – стрелка вправо и «ui_left» – стрелка влево.
  • «ui_right» \ «ui_left» \ «ui_up» \ «ui_down» – это одна из стандартных (предустановленных) входных команд Godot, которая обычно используется для навигации в UI (например, переключение кнопок интерфейса), но данные команды возможно использовать и для управления персонажем.

В данном случае при нажатии стрелки влево передается отрицательная скорость, а при нажатии стрелки вправо – положительная. Если не нажата ни одна из кнопок – персонаж остается на месте.

  • Последним этапом в данном скрипте будет добавление метода move_and_slide(). Это метод узла CharacterBody2D, который совершает перемещение с учетом коллизий (столкновений со стенами, платформами и т. п.), используя текущую velocity – скорость.
extends CharacterBody2D

var speed = 50

func _physics_process(_delta):
	if Input.is_action_pressed("ui_right"):
		velocity.x = speed
	elif Input.is_action_pressed("ui_left"):
		velocity.x = -speed
	else:
		velocity.x = 0

	move_and_slide()
Обратите внимание! Метод добавляется в тело функции, а не тело скрипта!

Самостоятельная работа!

На основе уже написанного кода попробуйте добавить перемещение по вертикали. Для этого можно использовать дополнительные проверки ввода!

Ниже ответ:

extends CharacterBody2D

var speed = 50

func _physics_process(_delta):

            if Input.is_action_pressed(«ui_right»):

                        velocity.x = speed

            elif Input.is_action_pressed(«ui_left»):

                        velocity.x = -speed

            else:

                        velocity.x = 0

            if Input.is_action_pressed(«ui_up»):

                        velocity.y = -speed

            elif Input.is_action_pressed(«ui_down»):

                        velocity.y = speed

            else:

                        velocity.y = 0

            move_and_slide()

Вторая условная конструкция проверяет стрелки вверх \ вниз и меняет скорость уже по вертикали, иначе персонаж стоит на месте.

По умолчанию в Godot в Input Map – списке действий заданы следующие действия:

  • ui_up – обычно назначена на клавишу «Стрелка вверх».
  • ui_down – обычно назначена на клавишу «Стрелка вниз».
  • ui_left – обычно назначена на клавишу «Стрелка влево».
  • ui_right – обычно назначена на клавишу «Стрелка вправо».

Вы можете посмотреть или изменить эти назначения, открыв список действий по пути: вкладка Проект → Настройки проекта → вкладка Список действий. В домашней работе вам потребуется использовать клавишу Shift, поэтому добавим ее сейчас в состав проекта.

Открыв список действий, как показано на рисунке 14, введите в поле название действия по аналогии с другими клавишами: ui_shift. После чего нажмите Добавить.

После того, как действие добавлено, ему необходимо назначить клавишу. Для этого нажмите кнопку +, как показано на рисунке 15.

В новом окне в разделе клавиши ввода найдите клавишу Shift (рис. 16). И нажмите ОК.

После чего появится запись – клавиша Shift будет соответствовать команде “ui_shift”.
