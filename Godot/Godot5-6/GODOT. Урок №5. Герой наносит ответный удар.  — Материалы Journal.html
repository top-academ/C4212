GODOT. Урок №5. Герой наносит ответный удар. 

Учебные материалы

Здравствуйте, юные разработчики игр! Рад снова всех видеть. Загружайте свои проекты, и пока они открываются, давайте быстро вспомним, с какими опасностями мы уже наполнили наш мир.

CharacterBody2D

Правильный ответ:

Вы отлично справились с созданием врагов! Но пока наш герой может только убегать и получать урон. Это нечестно! Сегодня мы исправим эту несправедливость. Наша миссия — дать герою оружие, научить его атаковать, а врагов — реагировать на его удары. А чтобы бой выглядел зрелищно, мы изучим волшебный инструмент Tween и создадим наши первые анимации без единого нарисованного кадра!

Для преподавателя: Какие способы атаки вы знаете в 2D-играх? Как визуально показать, что враг получает урон? 

Начнем с главного героя

Чтобы игрок мог атаковать, добавьте к корневому узлу главного героя узел Area2D, который будет появляться во время удара и проверять, есть ли в зоне атаки враги. Теперь структура игрока станет сложнее и будет выглядеть следующим образом:

Не забудьте дать новому узлу имя, например AttackZone. Внутри AttackZone добавьте узел CollisionShape2D и настройте зону атаки. Обязательно подключите сигнал body_entered к скрипту игрока main_hero.gd. Приступаем к модифицированию скрипта игрока – пропишем систему атаки. Для начала добавим переменные:

var attack_damage = 20
var is_attacking = false

Рассмотрим инициализацию:

  • var attack_damage = 20 — устанавливает значение урона атаки, который будет наноситься врагам.
  • var is_attacking = false – переменная-флаг, которая будет контролировать атаку, чтобы код «не ушел в себя»

Затем идет _on_attack_zone_body_entered(body) — этот метод позволяет точно определять момент атаки и контролировать область поражения, это основная функция, в которой будет прописываться вся логика атаки.

func _on_attack_zone_body_entered (body):
    if body.is_in_group("enemy") and not is_attacking:
        is_attacking = true
        body.take_damage(attack_damage)
        await get_tree().create_timer(0.5).timeout
        is_attacking = false

Когда AttackZone сталкивается с объектом (body), проверяем, находится ли этот объект в группе enemy, а также, что переменная-флаг выключена (false). Обратите внимание, что дальнейший код будет выполняться, только если оба условия будут выполнены! Поэтому не забудьте создать новую группу enemy через раздел Узел и назначить ее всем врагам!

Если это так, скрипт переключит переменную-флаг, чтобы персонаж не смог параллельно (или сразу после) запустить данную функцию еще раз, вызовет метод take_damage(), передавая значение урона attack_damage врагу, после чего запустит таймер на 0.5 секунды перед продолжением кода, и только затем обратно переключит флаг, чтобы функция смогла сработать еще раз.

Справка! Метод get_tree() возвращает главное дерево сцены – древо сцен и узлов. create_timer(0.5) создает новый таймер в дереве с продолжительностью 0.5 секунды. timeout — это сигнал, который срабатывает, когда таймер завершает свой отсчет. await приостанавливает выполнение текущей функции, пока не сработает timeout.

Как и в случае с системой жизни героя, враги теперь должны реагировать на наносимый персонажем урон. Для этого необходимо добавить метод take_damage(), который будет уменьшать их здоровье.

На примере статичного врага реализуем систему жизней и получения урона. Откройте скрипт EnemyStanding.gd. В первую очередь создайте переменную для показателя здоровья:

var health = 50

Затем добавьте функцию получения урона:

func take_damage(amount):
    health -= amount
    print("Здоровье врага:", health)
    if health <= 0:
        die()

Данная функция работает аналогично той, что была прописана в главном герое – в аргумент amount приходит значение урона от главного героя или других NPC и далее функция отнимает данное значение из переменной здоровья. В консоль выводится техническое сообщение, если же переменная жизни меньше нуля, либо равна ему, то вызывается функция die(), в которой описана логика смерти врага.

Остается добавить саму функцию die():

func die():
    queue_free()

Метод queue_free() ставит удаление в очередь, а Godot выполняет его безопасно в конце кадра, предотвращая возможные ошибки.

В нашей игре у персонажей нет покадровых анимаций, но это не значит, что мы не можем создать визуальные эффекты! Мы можем использовать изменение цвета, эффекты вспышек, плавные движения с помощью Tween и даже небольшие деформации спрайта.

Tween Это специальный инструмент в Godot, который создает плавные переходы (анимации) свойств объектов. Он позволяет изменять параметры (позицию, цвет, масштаб, прозрачность и др.) без использования покадровой анимации. Это простой и мощный способ создания анимаций без использования кадров.

Работа с данным инструментом не требует добавления отдельного узла в древо. Инструмент подключается через скрипт непосредственно в нужной функции. Рассмотрим вариант работы с инструментом на примере анимирования атаки главного героя. Вернитесь к скрипту main_hero.gd и добавьте новую функцию:

func attack():
var tween = create_tween()
tween.tween_property(self, "scale", Vector2(1.2, 1.2), 0.1)
tween.tween_property(self, "scale", Vector2(1, 1), 0.1)

Рассмотрим подробнее данную функцию func attack() – отвечает за анимацию атаки:

  • var tween = create_tween() – метод автоматически создает и управляет Tween, не требуя его в дереве сцены.
  • tween.tween_property(self, «scale», Vector2(1.2, 1.2), 0.1)  — строка, в которой происходит увеличение объекта:
  • self — объект, свойство которого будет анимироваться.
  • «scale» — имя свойства, которое будет меняться. «scale» отвечает за масштаб объекта, т.е. его размер.
  • Vector2(x, y) задает новый размер объекта по горизонтали и вертикали. 2 означает увеличение размера на 20%.
  • 1 — длительность анимации в секундах. Этот параметр задает время, за которое объект изменит свой масштаб. В данном случае 0.1 означает, что объект увеличится за 0.1 секунды.
  • tween.tween_property(self, «scale», Vector2(1, 1), 0.1) – затем происходит возвращение объекта к исходному состоянию

Чтобы данная анимация сработала, ее необходимо вызывать в ранее созданной функции _on_attack_zone_body_entered(body)

func _on_attack_zone_body_entered(body):
    if body.is_in_group("enemy") and not is_attacking:
        is_attacking = true
        attack()
        body.take_damage(attack_damage)
        await get_tree().create_timer(0.5).timeout
        is_attacking = false

Таким образом вместе с процессом атаки на врага будет запускаться анимация данного процесса!

Не оставим без внимания и врага! Вернитесь к скрипту статичного врага — EnemyStanding.gd. Здесь мы добавим дополнение к методу получения урона take_damage(amount).

func take_damage(amount):
    health -= amount
    print("Здоровье врага:", health)
    var tween = create_tween()
    tween.tween_property(self, "modulate", Color(1, 0.5, 0.5), 0.1)
    tween.tween_property(self, "modulate", Color(1, 1, 1), 0.1)
    if health <= 0:
        die()

Рассмотрим новые строки:

  • var tween = create_tween() – вновь создаем инструмент Tween.
  • tween.tween_property(self, «modulate», Color(1.5, 0.5, 0.5), 0.1) – в данном случае уже идет изменение не масштаба, а цвета. Свойство «modulate», отвечающее за цвет объекта позволяет менять цвета, создавать вспышки или эффекты удара, а аргумент Color(1, 1, 1) позволяет выбрать, какой именно цвет вы хотите применить в данном случае. Color(1.5, 0.5, 0.5) – красный оттенок, когда как три единицы – исходный цвет объекта. С помощью данных строк мы создаем эффект получения урона врагом. Четвертый параметр — 0.1 — длительность анимации в секундах. Этот параметр задает время, за которое объект изменит свой цвет.

В Godot класс Color(r, g, b, a) принимает четыре параметра:

  • r (красный) — от 0 (нет красного) до 1 (максимально насыщенный красный).
  • g (зеленый) — от 0 до 1.
  • b (синий) — от 0 до 1.
  • a (альфа-канал, прозрачность) — 0 (полностью прозрачный), 1 (полностью видимый). Данный параметр можно не указывать при использовании класса, как в примере выше. В таком случае будет принято значение по умолчанию — 1.

Примеры:

  • Color(1, 0, 0, 1) — Чистый красный.
  • Color(0, 1, 0, 1) — Чистый зеленый.
  • Color(0, 0, 1, 1) — Чистый синий.
  • Color(1, 1, 0, 1) — Желтый.
  • Color(1, 0.5, 0, 1) — Оранжевый.
  • Color(1, 1, 1, 0.5) — Полупрозрачный белый.

Таким образом для разных персонажей можно создавать разные вспышки при получении урона, что вносит яркое разнообразие в ваш проект!

Самостоятельная работа!

Необходимо добавить анимацию получения урона главному герою, а также всем типам врагов. Важно учесть, что главный герой теперь может атаковать всех врагов в ответ, поэтому у каждого из них должна быть внедрена система жизней и урона.

Дополнительный материал!

В Godot можно анимировать практически любое свойство узлов (Node2D, CharacterBody2D, Sprite2D, Label, Camera2D и т.д.) с помощью Tween. Давайте разберем основные свойства и варианты их использования.

№1: position – Позиция объекта (перемещение)

Тип: Vector2

Используется для: Перемещения объектов в пространстве.

Пример: Перемещение объекта в координаты (500, 300) за 1 секунду

var tween = create_tween()
tween.tween_property(self, "position", Vector2(500, 300), 1.0)

Чаще всего используется для плавного передвижения персонажей, врагов и объектов.

№2: scale – Масштаб (размер)

Тип: Vector2

Используется для: Изменения размера объекта (увеличение/уменьшение).

Пример: Увеличение в 1.5 раза за 0.5 секунды и возврат

var tween = create_tween()
tween.tween_property(self, "scale", Vector2(1.5, 1.5), 0.5)
tween.tween_property(self, "scale", Vector2(1, 1), 0.5)

Полезно для эффектов атаки, усиления, реакции на события (например, удар, телепорт, усиление персонажа).

№3: rotation_degrees – Вращение

Тип: float (градусы)

Используется для: Поворота объекта вокруг своей оси.

Пример: Поворот на 360 градусов за 1 секунду

var tween = create_tween()
tween.tween_property(self, "rotation_degrees", 360, 1.0)

Используется для вращения врагов, объектов, мечей, стрел, вращающихся ловушек и анимации атак.

№4: modulate – Цвет объекта

Тип: Color

Используется для: Изменения цвета, создания вспышек, эффектов удара.

Пример: Красная вспышка при уроне и возврат цвета

var tween = create_tween()
tween.tween_property(self, "modulate", Color(1.5, 0.2, 0.2), 0.1)  # Красный оттенок
tween.tween_property(self, "modulate", Color(1, 1, 1), 0.1)  # Возвращение цвета

Часто применяется при уроне, получении бонусов, исчезновении объектов.

№5: visible – Видимость объекта

Тип: bool (True/False)

Используется для: Показа и скрытия объектов.

Пример: Исчезновение объекта через 0.5 сек

await get_tree().create_timer(0.5).timeout
self.visible = false

Можно использовать, если объект не нужно удалять, а просто временно скрыть (например, сундуки, предметы, враги).

№6: global_position – Глобальная позиция

Тип: Vector2

Используется для: Перемещения объекта относительно всей сцены, а не локального родителя.

Пример: Перемещение объекта в глобальные координаты (800, 400)

var tween = create_tween()
tween.tween_property(self, "global_position", Vector2(800, 400), 1.0)

Используется, когда объект находится внутри другого узла и position работает неправильно.

№7: self_modulate – Прозрачность и цветовой оттенок

Тип: Color

Используется для: Изменения цвета и прозрачности (можно применять к группе объектов).

Пример: Постепенное исчезновение объекта

var tween = create_tween()
tween.tween_property(self, "self_modulate", Color(1, 1, 1, 0), 1.5)  # Прозрачность до 0

Полезно для эффектов призраков, дымки, плавного исчезновения врагов.

№8: light_energy – Интенсивность света (для Light2D)

Тип: float

Используется для: Изменения яркости источников света.

Пример: Имитация мигания света

var tween = create_tween()
tween.tween_property($Light2D, "energy", 0.5, 0.1)  # Свет тускнеет
tween.tween_property($Light2D, "energy", 1.0, 0.1)  # Свет восстанавливается

Используется в системах освещения, мигании факелов, эффекте вспышек.
