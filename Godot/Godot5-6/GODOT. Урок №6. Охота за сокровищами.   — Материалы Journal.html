Вы отлично помните, как устроить сражение! Но любое приключение становится еще интереснее, если в нем есть сокровища. Сегодня мы наполним наш мир ценными предметами: монетами для счета, зельями для лечения и бонусами для усиления. Мы создадим универсальный скрипт, который сможет превратить любой предмет во что угодно с помощью всего одной настройки в редакторе!

Для преподавателя: Какие виды предметов встречаются в roguelike-играх? Какие эффекты они могут давать? 

Рассмотрим примеры различных категорий предметов, которые могут вам встретиться в игре:

  1. Влияющие на счет
    • Монеты – увеличивают количество очков, которые можно использовать для покупок или определения прогресса.

2. Влияющие на здоровье

    • Зелья-аптечки – восстанавливают определенное количество здоровья.
    • Магические травы – дают постепенное восстановление HP на протяжении времени.
Справка! HP — аббревиатура от англ. health points — очки здоровья в компьютерных играх.

3. Влияющие на защиту

    • Щиты – временно уменьшают получаемый урон.
    • Доспехи – повышают максимальную защиту.

4. Влияющие на атаку

    • Мечи – временно увеличивают урон, наносимый врагам.
    • Ядовитые клинки – добавляют эффект отравления к ударам.

5. Бустеры и временные эффекты

    • Зелье ускорения – временно увеличивает скорость передвижения.
    • Свитки магии – могут давать случайные эффекты (усиление атаки, невидимость, защита и т. д.).
    • Амулеты – дают редкие пассивные бонусы, такие как авто-регенерация или двойной прыжок.

6. Сюжетные

    • Ключи, позволяющие открыть определенные двери

Итак, вы создали узел Coin1 для одной монетки в составе узла Coins. Расположили монетку на карте. Теперь необходимо прописать логику взаимодействия игрока с монеткой. Создайте новый скрипт — PickupItem.gd. Данный скрипт будет регулировать логику всех собираемых предметов, включая монетку.

Начнем создание скрипта с инициализации переменной:

@export var item_type = "coin"

Объявляем переменную item_type, которая определяет тип предмета (coin, health, shield, speed). @export позволяет изменять значение переменной в редакторе, в Инспекторе, не редактируя код (рис. 1).

Таким образом при привязке скрипта к разным объектам, необходимо в Item Type указывать тип предмета.

Теперь необходимо прописать логику пересечения с другим объектом:

func _on_body_entered(body):
    if body.is_in_group("player"):
        apply_effect(body)
        animate_and_remove()

Рассмотрим функцию:

if body.is_in_group(«player»): проверяет, является ли body игроком (так как у игрока есть группа player). Если body — это игрок, вызываются две функции:

  • apply_effect(body) — дает игроку эффект предмета (например, добавляет очки или здоровье).
  • animate_and_remove() — запускает анимацию исчезновения предмета.

Далее добавим применение эффекта предмета, в данном случае – начисление очков за собранную монету:

func apply_effect(player):
    match item_type:
        "coin":
            player.add_score(10)

Функция apply_effect(player) обрабатывает эффекты предметов в зависимости от их типа — item_type. Чтобы проверить значение item_type, используется оператор match. Если значение «coin», то игроку должно быть начислено 10 очков!

Рассмотрим теперь вторую функцию — animate_and_remove() — чтобы создать эффект плавного исчезновения после подбора, вам нужно вновь использовать Tween для анимации прозрачности (свойство modulate).

func animate_and_remove():
    var tween = create_tween()
    tween.tween_property(self, "modulate:a", 0.0, 0.5)
    await tween.finished
    queue_free()

Рассмотрим данную функцию построчно:

  • create_tween(): создаёт объект для анимации.

  • tween_property(): плавно изменяет свойство modulate (цвет и прозрачность) до значения 0.0 (полностью прозрачный).

  • await tween.finished: ожидает, пока анимация не закончится.

  • queue_free(): удаляет объект только после того, как он станет полностью невидимым.

Однако! Вспомним, что на данный момент у главного героя нет функции add_score(), это означает, что скрипт монетки будет выдавать ошибку и закрывать игру. Поэтому необходимо обновить скрипт главного героя — main_hero.gd.

В первую очередь необходимо добавить новую переменную:

var score = 0

Переменная, которая будет хранить в себе количество очков.

Теперь необходимо добавить саму функцию:

func add_score(amount):
    score += amount
    print("Ваш счет: ", score)

Данная функция добавляет в переменную score количество очков, которое пришло в параметр amount. Команда print выводит в консоль техническое сообщение о текущем количестве очков.

Теперь остается добавить анимацию «парения» собираемому объекту, чтобы выделить его для героя. Создадим функцию func _ready() в скрипте PickupItem.gd и добавим новый код – бесконечный цикл, в котором будет работать анимация:

func _ready():
    while true:
        var tween = create_tween()
        tween.tween_property(self, "position:y", position.y - 5, 0.5)
        await tween.finished
        tween = create_tween()
        tween.tween_property(self, "position:y", position.y + 5, 0.5)
        await tween.finished

Разберем подробнее:

  • var tween = create_tween() — создается новый Tween, который будет управлять анимацией объекта. Каждый Tween создается заново перед каждым движением, чтобы избежать ошибки зависания.
  • tween_property(self, «position:y», position.y — 5, 0.5) – сообщает о том, что объекту необходимо уменьшить координату y на 5 пикселей за 0.5 секунды. В результате предмет поднимается вверх.
  • await tween.finished — ожидание, пока анимация завершится. Код не продолжает выполнение, пока Tween не закончит движение.

Затем Tween пересоздается, иначе анимация дальше работать не будет

  • tween.tween_property(self, “position:y”, position.y + 5, 0.5) — теперь объект движется обратно вниз на 5 пикселей за 0.5 секунды. Таким образом, предмет возвращается в исходное положение.

После чего завершается уже эта часть анимации. Таким образом создаются парящие объекты!

Для функции apply_effect(player) скрипта PickupItem.gd добавим «ветки» для других типов объектов:

func apply_effect(player):
    match item_type:
        "coin":
            player.add_score(10)
        "health":
            player.heal(20)
        "shield":
            player.add_shield(10)
        "speed":
            player.boost_speed(5)
        "power":
            player.add_power(3)
  • Если item_type == «coin», вызывается add_score(10). Это добавляет 10 очков игроку.
  • Если item_type == «health», вызывается heal(20). Игрок получает 20 единиц здоровья.
  • Если item_type == «shield», вызывается add_shield(10). Игрок получает 10 единиц дополнительной защиты.
  • Если item_type == «speed», вызывается boost_speed(5). Ускоряет игрока на 5 секунд.
  • Если item_type == «power», вызывается add_power(3). Игрок получает усиление атаки на 3 единицы.

Подведем небольшой вывод:

  • apply_effect() определяет, какой эффект получит игрок, исходя из item_type.
  • Используется match, чтобы избежать множества if-else.
  • Этот метод позволяет легко добавлять новые предметы, просто дописав новую строку.

Теперь необходимо дополнить скрипт главного героя — main_hero.gd, чтобы он смог принять эффекты, которые ему передают предметы:

func heal(amount):
	health += amount
	print("Здоровье: ", health)

func add_shield(amount):
	shield += amount
	print("Защита: ", shield)

func boost_speed(duration):
	speed += 50
	await get_tree().create_timer(duration).timeout
	speed -= 50

func add_power(amount):
	attack_damage += amount
	print("Сила атаки: ", shield)

Рассмотрим данные функции:

  • func add_score(amount) — увеличивает счет игрока на amount. score += amount — прибавляет переданное значение к текущему счету. print(«Счет:», score) — выводит текущий счет в консоль (для отладки).
  • func heal(amount) — увеличивает здоровье игрока. health += amount — прибавляет amount к текущему health. print(«Здоровье:», health) — выводит в консоль текущее количество здоровья.
  • func add_shield(amount) — увеличивает защиту игрока. shield += amount — добавляет amount к текущему значению защиты. print(«Защита:», shield) — выводит в консоль текущую защиту.
  • func boost_speed(duration) — временно увеличивает скорость игрока. speed += 50 — прибавляет 50 единиц скорости. await get_tree().create_timer(duration).timeout — ждет указанное duration секунд перед выполнением следующей строки. speed -= 50 — возвращает скорость к обычному значению после истечения времени.
  • func add_power(amount) — увеличивает урон игрока. attack_damage += amount — прибавляет amount к урону. print(«Сила атаки:», attack_damage) — выводит в консоль новый уровень атаки.
