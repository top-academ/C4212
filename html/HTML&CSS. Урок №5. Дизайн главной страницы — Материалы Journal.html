Понятия шаблона сайта

При формировании нового сайта не всегда приходится начинать разработку с нуля. Иногда для создания веб-страницы достаточно воспользоваться заранее подготовленным шаблоном. 

Шаблон — это готовый html-код страницы, который можно использовать для создания разных сайтов. Чтобы превратить шаблон в уникальную страницу, достаточно добавить уникальный контент (текст, изображения) и настроить стили (рис. 1).

Шаблоны — отличный вариант как для начинающих разработчиков, так и для опытных специалистов. Чаще всего шаблоны используют в небольших проектах, например, при создании сайта-визитки или блога. Все шаблоны можно разделить на два вида:

Бесплатные

Большинство бесплатных шаблонов содержит основные элементы страницы. Отличается только тематика и расположение контента. Выбрав подобный тип шаблона, пользователь может самостоятельно изменять любые части кода, будь то HTML-код страницы или стили.
Например: https://freehtmlthemes.ru/ (рис. 2)

Платные

Техническая поддержка — главная особенность платных шаблонов. Разработчики подобных сайтов часто помогают настраивать и изменять такие шаблоны, а также дают подсказки по поводу работы тех или иных функций.

Также платные шаблоны более уникальны, поэтому встретить в сети сайт с таким же дизайном намного сложнее. Ресурсов, предоставляющих платные шаблоны, так же много, как и бесплатных

Блочная верстка шаблона. Стилизация шаблона

Шаблоны, как и сайты, строятся из блоков. При блочной верстке сайт разбивается на несколько основных частей, которые содержат те или иные элементы управления, контент, всевозможные формы ввода, кнопки и т.д. Основные элементы практически любого сайта — это хедер, навигационная панель, контентный блок и футер. Остальные элементы зачастую добавляются внутрь этих блоков.

Рассмотрим расположение блоков на реальном примере. Для этого откроем любой шаблон, например, Fantom: https://colorlib.com/wp/template/fantom/. 

Даже не заглядывая в код, можно убедиться, что данный сайт состоит из блоков (рис. 3)

Откроем в браузере код сайта (горячие клавиши: CTRL+SHIFT+I). Наведя курсор на любой из вышеперечисленных элементов, можно увидеть его границы. К примеру, блок <header> содержит логотип и навигационное меню. В блоке <section> размещены слайдеры, статьи и прочий контент сайта. Последний блок — <footer>. Он содержит краткую информацию о владельце сайта, форму Newsletter для подписки на обновления, а также ссылки на социальные сети. Стоит отметить, что в футере иногда дублируют пункты главного меню. Это делается для того, чтобы пользователю не пришлось пролистывать вверх весь сайт, когда он доберется до конца страницы (видео 1).

Чтобы превратить шаблон в полноценный сайт, его необходимо стилизовать. Для этого следует задать основные цвета, изменить текст и изображения, а также добавить логотип.

Дополнительно можно изменить отдельные блоки кода, к примеру, по-другому разбить страницы или поменять последовательность подачи материала.

Создание блога

Давайте создадим шаблон блога, посвященного вселенной LEGO, и наполним его контентом. Шаблон должен состоять из хедера, навигационной панели, контентного блока, сайдбара и футера. 

Оформление главной страницы с помощью блоков

Ссылка на готовый проект в CodePen: https://codepen.io/Academy-top/pen/bGQBQpa?editors=1100

Главная страница сайта будет построена с помощью блоков. Для начала создадим «скелет» страницы, который и будем использовать в качества шаблона. Работать с кодом будем внутри тегов <body>. Таким образом, структура будет следующей: хедер, основная часть, футер.

<!-- HEADER -->
<header>
 …
</header>
<!-- MAIN -->
<main>
 …
</main>
<!-- FOOTER -->
<footer>
 …
</footer>

Комментарии <!— … —> нужны, чтобы в дальнейшем было проще ориентироваться разработчику. Особенно это важно, если предполагается, что шаблон будет использован для создания новых страниц, например, страницы с постом или контактами. Представьте, что спустя некоторое время вам вновь нужно будет вернуться к проекту. Комментарии делают код более понятным и читабельным.

Создание логотипа сайта

Прежде чем приступать к разработке страницы, давайте придумаем уникальный логотип для сайта. 

Чтобы сделать крутой логотип для блога, следует подумать о хобби и увлечениях владельца сайта. Логотип должен характеризовать тематику сайта, но при этом не содержать множество мелких деталей. К примеру, для логотипа блога фотографа можно использовать стилизованное изображение фотокамеры, для любителя литературы подойдет логотип с изображением книги, а для путешественника — со стилизованной машиной.

Придерживаясь выбранной тематики сайта, давайте создадим логотип для LEGO-блога. Для этого воспользуемся сервисом Flyvi: https://flyvi.io/ru. 

Создадим новый документ в Flyvi. Основу логотипа можно самостоятельно нарисовать в любом графическом редакторе. 

Она должна быть максимально простой и раскрывать идею сайта (рис. 4)

Готовую картинку нужно скачать на компьютер и перенести в папку img, затем изображение можно загрузить в облачное хранилище Postimage (https://postimages.org/)  для использования в проекте CodePen. Обратите внимание, если вы еще не определились с дизайном или цветовой палитрой сайта, можно создать несколько вариантов логотипа.

Хедер

Приступим к созданию шапки сайта. Она будет определена тегами <header>. Эту часть зальем голубым цветом, а слева добавим созданный ранее логотип (рис. 5)

<header>
    <img src=”https://i.postimg.cc/bJptxPg6/transformed.png”
     width=”200” draggable=”false” alt=”Logo”>
</header>

Футер

Футер — обязательный элемент сайта. Он может содержать информацию об авторе, меню в развернутом виде, контакты и ссылки на социальные сети.

Теперь создадим футер. Его условно можно разделить на четыре части: логотип, колонка с информацией об авторе, колонка с контактами автора и копирайт. Для колонок мы создадим <div class=»footer-columns»> и поместим в него два <div class=»footer-column»> (рис. 6). 

<footer>
  <div>
    <div>
      <div>
        <img decoding="async" src="https://i.postimg.cc/bJptxPg6/transformed.png" alt="Logo" width="250">
      </div>
      <div class="footer-columns">
        <div class="footer-column">
          <h3>О нас</h3>
          <p>
            Пока дети строят свои собственные миры из кубиков LEGO, мы играем свою роль в том, чтобы оказывать положительное влияние на мир, в котором они живут сегодня и который унаследуют в будущем.
          </p>
        </div>
        <div class="footer-column">
          <h3>Свяжитесь с нами</h3>
          <p>
            Если вы заинтересованы в работе со мной или просто хотите поздороваться, просто напишите по электронной почте.
            <br><strong>Email: <a href="#">contact@mail.com</a></strong>
          </p>
          </div>
      </div>
    </div>
    <center>
      <small>Copyright &copy; 2025</small>
    </center>
  </div>
</footer>

Для корректного отображения колонок зададим следующие стили:

.footer-columns {
	 display: flex;
}
.footer-column {
	 flex: 50%;
margin: 20px;
}

А для header и footer сделаем заливку голубым цветом и внутренний отступ в 10 пикселей:

header, footer {
	 background-color: #9debf5;
padding: 10px;
}

Между header и footer создадим блок main. Он будет содержать посты блога. Сделаем его фон серебряного цвета, а фон постов — белым. Изображения постов разместим слева от текста, используя display: flex. Создадим один пост (рис. 7):

<main>
<div>
  <div class="post">
    <img decoding="async" height="230" src="https://i.postimg.cc/QxfCP5bJ/2023-06-24-22-43-17.png" alt="Card image cap">
    <div>
      <h3>Соник Супер-ежик</h3>
      <p>
        Откройте для себя один из самых запоминающихся уровней видеоигр: легендарные Зеленые холмы ежика Соника. Знакомые персонажи — минифигурка ежа Соника LEGO®, а также фигурки доктора Эггмана со сборным эггмобилем, мотожука с двумя вариантами лица и крабмита.
      </p>
      <a href="#">Подробнее</a>
    </div>
  </div>

Укажем следующие стили:

main {
padding: 10px;
background-color: silver;
}
.post {
background-color: white;
padding: 20px;
border-radius: 10px;
margin-bottom: 20px;
	 display: flex;
}
.post img {
margin: 20px;
}

Теперь, чтобы добавить еще посты, просто скопируем <div class=»post»> и поменяем ссылку на изображение. В результате у нас получился вот такой блог (видео 2)
